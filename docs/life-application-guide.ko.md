# 초보자용 생활 적용 가이드

바이브 코딩을 전혀 몰라도 이 시스템을 쓸 수 있어야 합니다. 핵심은 "AI에게 코드를 짜게 한다"가 아니라 "내 자료를 정리하는 직원처럼 일하게 한다"입니다.

## 1. 처음 만들기 좋은 위키 5가지

### 내 업무 노하우 위키

넣을 자료:

- 회의록
- 제안서 초안
- 고객 질문
- 반복 업무 매뉴얼

질문 예:

```text
최근 고객 질문을 기준으로 내가 자주 설명하는 개념 5개를 뽑고,
각 개념 페이지를 업데이트해줘.
```

### 공부 위키

넣을 자료:

- 강의 자막
- 책 요약
- 논문 초록
- 내가 직접 푼 문제

질문 예:

```text
이번 주에 넣은 공부 자료를 기준으로 아직 헷갈리는 개념을 찾아줘.
개념 페이지와 다음 학습 질문을 만들어줘.
```

### 콘텐츠 아이디어 위키

넣을 자료:

- 유튜브 자막
- 스레드 초안
- 블로그 초안
- 독자 반응

질문 예:

```text
내가 반복해서 말하는 주제를 찾아서 콘텐츠 기둥 5개로 정리해줘.
각 기둥마다 근거가 되는 source를 연결해줘.
```

### 개인 회고 위키

넣을 자료:

- 일기
- 주간 회고
- 목표 메모
- 실패/성공 기록

질문 예:

```text
최근 회고에서 반복되는 문제 패턴을 찾아줘.
감정 기록과 실제 행동을 구분해서 wiki에 정리해줘.
```

### 프로젝트 판단 위키

넣을 자료:

- 결정 기록
- 비교표
- 리서치 자료
- 실험 결과

질문 예:

```text
이 프로젝트에서 이미 결정된 것과 아직 열린 질문을 분리해줘.
모순되는 결정이 있으면 contradiction-candidate로 표시해줘.
```

## 2. 첫 30분 실습

### 0-5분: 폴더 복사

패키지 루트 안에서 `templates/llm-wiki`를 복사해서 `my-life-wiki`처럼 이름을 바꿉니다.

`scripts/` 폴더는 reasoner/lint 검사 도구이므로 지우지 않습니다.

### 5-10분: Obsidian에서 열기

Obsidian에서 복사한 폴더를 vault로 엽니다.

### 10-15분: raw에 자료 하나 넣기

처음에는 짧은 글 하나만 넣습니다.

좋은 첫 자료:

- 오늘 회의록
- 내가 쓴 글 한 편
- 유튜브 자막 일부
- 공부 노트 한 장

### 15-25분: AI에게 ingest 시키기

```text
CLAUDE.md 규칙을 읽고 raw/example-source.md 대신 내가 넣은 자료를 ingest 해줘.
wiki 페이지와 ontology/abox.json을 업데이트하고,
마지막에 reasoner와 lint 검사를 실행해줘.
```

### 25-30분: 질문하기

```text
방금 만든 wiki를 기준으로 내가 다음에 물어봐야 할 질문 5개를 제안해줘.
좋은 질문은 wiki/questions/에 저장해줘.
```

## 3. 초보자가 피해야 할 실수

- 처음부터 자료 100개를 넣지 마세요.
- 처음부터 graph DB나 vector DB를 붙이지 마세요.
- AI가 만든 위키를 전부 믿지 말고 reasoner/lint를 돌리세요.
- 출처 없는 주장을 그대로 남기지 마세요.
- 위키 페이지를 사람이 직접 다 쓰려고 하지 마세요.

## 4. 좋은 운영 감각

이 시스템에서 사람의 역할은 자료를 고르고, 질문을 잘 던지고, 중요한 판단을 승인하는 것입니다.

AI의 역할은 반복 정리, 교차 링크, 누락 탐지, 모순 후보 표시, 형식 유지입니다.

처음 성공 기준은 작게 잡으세요.

```text
자료 3개를 넣었고,
개념 페이지 5개가 생겼고,
reasoner 오류가 0개이고,
내가 다시 찾을 수 있는 질문 페이지가 1개 생겼다.
```
